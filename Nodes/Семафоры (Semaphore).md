## 1. Семафоры System V

API System V оперирует **наборами семафоров**, что позволяет работать с группой семафоров как с единым объектом.

### 1.1. `ftok()` — генерация ключа IPC

```c unfold
key_t ftok(const char *pathname, int proj_id);
```
**Параметры**

- `pathname` — путь к существующему файлу. Используются метаданные файла (устройство, inode) для создания ключа.
    
- `proj_id` — идентификатор проекта (8 бит). Позволяет из одного файла получить несколько различных ключей.
    

**Возвращаемое значение**  
Ключ типа `key_t` (используется в `semget`). При ошибке возвращается `(key_t)-1`.

---

### 1.2. `semget()` — создание или получение доступа к набору семафоров

```c unfold
int semget(key_t key, int nsems, int semflg);
```

**Параметры**

- `key` — ключ IPC (обычно полученный через `ftok`, либо `IPC_PRIVATE` для создания приватного набора).
    
- `nsems` — количество семафоров в наборе. При создании должно быть ≥1.
    
- `semflg` — флаги и права доступа. Обязательные комбинации:
    
    - `IPC_CREAT` — создать набор, если он не существует.
        
    - `IPC_CREAT | IPC_EXCL` — создать новый набор; если уже существует — ошибка `EEXIST`.
        
    - Права доступа, например `0644`.
        
    - Для подключения к существующему набору достаточно передать `0`.
        

**Возвращаемое значение**  
Идентификатор набора семафоров (`semid`) при успехе, `-1` при ошибке.

**Примечания**  
После создания нового набора **значения семафоров не определены**. В Linux они инициализируются нулями, но для переносимости следует явно задать начальные значения через `semctl()`.

---

### 1.3. `semop()` — выполнение операций над семафорами

```c unfold
int semop(int semid, struct sembuf *sops, size_t nsops);
```

**Параметры**

- `semid` — идентификатор набора, полученный из `semget`.
    
- `sops` — указатель на массив структур `sembuf`, каждая описывает операцию над одним семафором.
    
- `nsops` — количество элементов в массиве.
    

**Структура `sembuf`**

```c unfold
struct sembuf {
    unsigned short sem_num; /* индекс семафора в наборе (0..nsems-1) */
    short          sem_op;  /* операция: положительное число (+), отрицательное (-), 0 */
    short          sem_flg; /* флаги: 0, IPC_NOWAIT, SEM_UNDO */
};
```

**Операции (`sem_op`)**

- **`sem_op > 0`** — увеличить значение семафора на `sem_op`. Операция никогда не блокирует.
    
- **`sem_op = 0`** — проверка на равенство нулю:
    
    - если значение семафора равно 0, операция завершается немедленно;
        
    - если значение не 0, процесс блокируется до тех пор, пока семафор не станет равным 0.
        
- **`sem_op < 0`** — попытка уменьшить значение семафора на `-sem_op`:
    
    - если текущее значение ≥ `-sem_op`, операция выполняется (значение уменьшается);
        
    - если значение недостаточно, процесс блокируется до тех пор, пока оно не станет достаточным.
        

**Флаги (`sem_flg`)**

- `0` — стандартное поведение (блокировка при необходимости).
    
- `IPC_NOWAIT` — не блокироваться; если операция должна быть заблокирована, вызов возвращает ошибку `EAGAIN`.
    
- `SEM_UNDO` — автоматически отменить операцию при завершении процесса (помогает избежать deadlock'ов).
    

**Возвращаемое значение**  
`0` при успехе, `-1` при ошибке.

**Примечания**  
Все операции в массиве `sops` выполняются **атомарно**. Если хотя бы одна операция не может быть выполнена немедленно, процесс блокируется до тех пор, пока не станет возможным выполнить все одновременно.

---

### 1.4. `semctl()` — управление набором семафоров

```c unfold
int semctl(int semid, int semnum, int cmd, ...);
```

**Параметры**

- `semid` — идентификатор набора.
    
- `semnum` — номер семафора (используется не для всех команд).
    
- `cmd` — команда управления.
    
- `...` — необязательный аргумент, обычно `union semun` (определяется пользователем).
    

**Основные команды**

- `IPC_STAT` — получить информацию о наборе в структуру `struct semid_ds`.
    
- `IPC_SET` — установить параметры набора.
    
- `IPC_RMID` — удалить набор семафоров.
    
- `GETVAL` — получить значение одного семафора (индекс `semnum`).
    
- `SETVAL` — установить значение одного семафора (индекс `semnum`); **необходимо для инициализации**.
    
- `GETALL` — получить значения всех семафоров в массив.
    
- `SETALL` — установить значения всех семафоров из массива.
    

**Возвращаемое значение**  
Зависит от команды: обычно `0` при успехе, для `GETVAL` — текущее значение семафора; при ошибке `-1`.

---

### 1.5. Пример: бинарный семафор (мьютекс) через System V

Для одного семафора в наборе.

**Блокировка (`lock`)**

```c unfold
struct sembuf ops[2];
ops[0].sem_num = 0; ops[0].sem_op = 0; ops[0].sem_flg = 0; // ждём, пока станет 0
ops[1].sem_num = 0; ops[1].sem_op = 1; ops[1].sem_flg = 0; // затем увеличиваем на 1
semop(semid, ops, 2);  // атомарно
```

**Разблокировка (`unlock`)**

```c unfold
struct sembuf op = {0, -1, 0};  // уменьшаем на 1
semop(semid, &op, 1);
```

## 2. Семафоры POSIX

POSIX предоставляет два типа семафоров: **именованные** (видимые через файловую систему) и **разделяемые в памяти** (неименованные). Ниже описаны именованные, так как они наиболее близки к System V.

### 2.1. `sem_open()` — создание или открытие именованного семафора

```c unfold
sem_t *sem_open(const char *name, int oflag, /* mode_t mode, unsigned int value */);
```

**Параметры**

- `name` — строковое имя семафора. **Должно начинаться с '/'** (например, `"/mysem"`).
    
- `oflag` — флаги:
    
    - `0` — открыть существующий семафор.
        
    - `O_CREAT` — создать, если не существует. При этом требуются два дополнительных аргумента:
        
        - `mode` — права доступа (например, `0644`);
            
        - `value` — начальное значение семафора (должно быть ≥0).
            
    - `O_CREAT | O_EXCL` — создать новый семафор; если уже существует — ошибка `EEXIST`.
        

**Возвращаемое значение**  
Указатель на объект типа `sem_t` при успехе, `SEM_FAILED` при ошибке.

**Примечания**  
Именованные семафоры хранятся в специальной файловой системе (обычно `tmpfs`, монтируется в `/dev/shm`). Они сохраняются до явного удаления.

---

### 2.2. `sem_close()` — закрытие семафора

```c unfold
int sem_close(sem_t *sem);
```

**Параметры**

- `sem` — указатель на семафор, полученный из `sem_open`.
    

**Возвращаемое значение**  
`0` при успехе, `-1` при ошибке.

**Примечания**  
Закрывает семафор для текущего процесса, уменьшает счётчик ссылок. Сам объект не удаляется из системы.

---

### 2.3. `sem_unlink()` — удаление именованного семафора

```c unfold
int sem_unlink(const char *name);
```
**Параметры**

- `name` — имя семафора.
    

**Возвращаемое значение**  
`0` при успехе, `-1` при ошибке.

**Примечания**  
Помечает семафор на удаление. Фактическое удаление происходит после того, как все процессы закроют его (вызовут `sem_close` или завершатся).

---

### 2.4. `sem_post()` — увеличение семафора

```c unfold
int sem_post(sem_t *sem);
```

**Параметры**

- `sem` — указатель на семафор.
    

**Возвращаемое значение**  
`0` при успехе, `-1` при ошибке.

**Примечания**  
Атомарно увеличивает значение семафора на 1. Если есть процессы, ожидающие на `sem_wait`, один из них разблокируется. Каждый вызов `sem_post` атомарен по отношению к другим вызовам для того же семафора, но последовательность нескольких вызовов не является атомарной (между ними может произойти переключение контекста).

---

### 2.5. `sem_wait()` — уменьшение семафора (блокирующее)

```c unfold
int sem_wait(sem_t *sem);
```

**Параметры**

- `sem` — указатель на семафор.
    

**Возвращаемое значение**  
`0` при успехе, `-1` при ошибке.

**Примечания**  
Атомарно уменьшает значение семафора на 1. Если текущее значение равно 0, процесс блокируется до тех пор, пока значение не станет положительным (т.е. пока другой процесс не вызовет `sem_post`), после чего уменьшает и продолжает.

---

### 2.6. `sem_trywait()` — неблокирующая попытка уменьшить семафор

```c unfold
int sem_trywait(sem_t *sem);
```

**Параметры**

- `sem` — указатель на семафор.
    

**Возвращаемое значение**  
`0` при успехе; `-1` с `errno == EAGAIN`, если семафор занят (значение равно 0).

**Примечания**  
Неблокирующий аналог `sem_wait`. Если значение положительное, уменьшает его и возвращает 0; иначе возвращает ошибку.

---

### 2.7. `sem_getvalue()` — получение текущего значения семафора

```c unfold
int sem_getvalue(sem_t *sem, int *sval);
```

**Параметры**

- `sem` — указатель на семафор.
    
- `sval` — указатель на переменную, куда будет записано значение.
    

**Возвращаемое значение**  
`0` при успехе, `-1` при ошибке.

**Примечания**  
Возвращает текущее значение семафора. Однако значение может быть устаревшим сразу после получения, так как другой процесс может его изменить. В Linux для ожидающих процессов может возвращать отрицательное число по модулю, но стандарт требует неотрицательное значение.

---

### 2.8. Пример: бинарный семафор POSIX

```c unfold
sem_t *sem = sem_open("/mysem", O_CREAT, 0644, 1);  // инициализация значением 1
// Блокировка
sem_wait(sem);
// Критическая секция
// Разблокировка
sem_post(sem);
// Удаление при завершении
sem_close(sem);
sem_unlink("/mysem");
```

## 3. Реализация в ядре Linux

В исходном коде ядра (`kernel/semaphore.c`) семафоры реализованы с использованием спин-блокировки для защиты счётчика и очереди ожидания.  
Упрощённая логика функции `down()` (аналог `sem_wait`):

```c unfold
void down(struct semaphore *sem)
{
    unsigned long flags;
    raw_spin_lock_irqsave(&sem->lock, flags);
    if (likely(sem->count > 0))
        sem->count--;
    else
        __down(sem);  // помещает текущий процесс в очередь ожидания
    raw_spin_unlock_irqrestore(&sem->lock, flags);
}
```

Функция `up()` (`sem_post`) может вызываться из любого контекста, включая обработчики прерываний (в отличие от мьютексов).

## 4. Сравнение System V и POSIX семафоров

| Характеристика           | System V                                              | POSIX (именованные)                                      |
| ------------------------ | ----------------------------------------------------- | -------------------------------------------------------- |
| **Идентификация**        | Целочисленный ключ (`key_t`), полученный через `ftok` | Строковое имя, начинающееся с `/`                        |
| **Объект синхронизации** | Набор семафоров (массив)                              | Один семафор                                             |
| **Инициализация**        | Требуется явный вызов `semctl` после `semget`         | Задаётся при `sem_open` (параметр `value`)               |
| **Операции**             | Атомарные группы операций над несколькими семафорами  | Только одиночные операции                                |
| **Производительность**   | Всегда системные вызовы                               | Может быть оптимизирована (user-space для неименованных) |
| **Удаление**             | `semctl(IPC_RMID)`                                    | `sem_unlink()`                                           |
| **Сброс при завершении** | Флаг `SEM_UNDO` в `semop`                             | Не предусмотрен (только явный `sem_post`)                |
