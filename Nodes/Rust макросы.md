Макросы нужно сначала определить, и только потом использовать. Определять их позднее нельзя.

Чтобы использовать макрос из другого модуля (скоупа) нужно вставить `#[macro_use]`.
```rust  title="Простой пример макроса"
#[macro_use]
mod macros {
    macro_rules! my_macro {
        () => {
            println!("Print my name");
        };
    }
}

fn main() {
    my_macro!();
}

```

***`#[macro_use]`***
- **Назначение**: обычно используется для **импорта макросов из внешних крейтов** (например, `#[macro_use] extern crate log;`).
- В контексте **перед собственным макросом** (внутри крейта) этот атрибут раньше использовался, чтобы макрос был доступен во всём текущем модуле без явного `use`.
- **Современный Rust**: этот способ считается устаревшим. Лучше явно импортировать макросы через `use crate::macro_name;` или полный путь. Тем не менее, старый код может содержать `#[macro_use]` перед объявлением макроса для совместимости.

```rust title="Пример макроса с 0 и 1 аргументами"
#[rustfmt::skip]
macro_rules! my_macro {
    () => {
        println!("Check out of my macro!");
    };
    ($val:expr) => {
        println!("Look at this other macro: {}", $val);
    }
}

fn main() {
    my_macro!();
    my_macro!(777);
}
```

***`#[rustfmt::skip]`***
* Запрещает форматеру rust форматировать блок ниже

```rust title="Как реализовать макрос вектора"
#[macro_export]
macro_rules! my_vec {
    ( $( $x:expr), * ) => {
        {
            let mut temp_vec = Vec::new();
            $ (
                temp_vec.push($x);
            )*
            temp_vec
        }
    };
}

fn main() {
    let a = my_vec!(1, 2, 3);
}
```

***`#[macro_export]`***
- **Назначение**: **экспортировать макрос** из текущего крейта, чтобы другие крейты могли его использовать.
- Без этого атрибута макрос остаётся приватным (виден только внутри вашего крейта).
- Дополнительно можно указать `#[macro_export(local_inner_macros)]` для особых случаев (чтобы внутренние вызовы макросов тоже разрешались локально).
